## 🎯 빠른 시작 (Quick Start)

### 👔 경영진/비즈니스 담당자용 (5분 읽기)

**이 시스템이 하는 일은?**

```
매출 데이터를 자동 분석하여 비즈니스 인사이트를 제공합니다.

입력: 매출 데이터 (CSV 파일)
     ↓
처리: 자동화된 데이터 분석 (클릭 한 번)
     ↓
출력: 
  • 시각 자료 (차트, 그래프)
  • 통계 분석 (지역별 차이)
  • 판매 예측 모델
  • 자세한 보고서
```

**핵심 가치**

| 항목 | 효과 |
|------|------|
| ⏱️ **속도** | 수동 분석 (수일) → 자동 분석 (10초) |
| 💰 **비용** | 분석가 고용 비용 절감 |
| 📊 **정확도** | 수학적 검증 (통계 검정) |
| 🔍 **투명성** | 모든 작업 자동 기록 (감시 추적성) |
| 🎯 **의사결정** | 데이터 기반 의사결정 지원 |

**의사결정을 위한 3가지 핵심 결과**

1. **지역별 판매 현황** (인터랙티브 차트)
   - 어느 지역이 가장 잘 팔리는가?
   - 실시간 상호작용 가능한 차트로 확인

2. **카테고리별 전략** (통계 분석)
   - 각 지역에서 어떤 상품이 잘 팔리는가?
   - 통계적으로 검증된 연관성

3. **미래 판매 예측** (머신러닝 모델)
   - 다음 달 예상 매출은?
   - 자동으로 학습된 예측 모델 제공

**필요한 액션**

```
✅ 매월 sales_100k.csv 파일 제공
✅ 2-3분 기다리기
✅ 분석 결과 확인 및 의사결정
```

## 📊 기능 상세 설명

### 1️⃣ EDA (Exploratory Data Analysis) - 탐색적 데이터 분석

**하는 일**: 데이터의 특징을 시각적으로 파악

```
┌─────────────────────────────────┐
│   2x2 시각화 대시보드 생성      │
├─────────────────────────────────┤
│ ① 히스토그램 + KDE              │ → 매출액 분포 형태
│   (왜도, 첨도 파악)             │
│                                 │
│ ② 박스플롯                      │ → 이상치 탐지
│   (카테고리별 분포)             │   (이상 거래 찾기)
│                                 │
│ ③ 라인 차트                     │ → 시간 변화 추이
│   (월별 매출 추이)              │   (계절성 파악)
│                                 │
│ ④ 히트맵                        │ → 변수 간 관계
│   (상관계수)                    │   (강한 영향 요소)
└─────────────────────────────────┘
```

**얻는 인사이트**
- 📈 매출 분포는 우상향 비대칭 (소수의 고액 거래)
- 🎯 카테고리별 중앙값은 유사하나 이상치 많음
- 📅 월별 매출은 큰 변동성 (계절성 존재)
- 🔗 수량과 단가가 매출과 강한 상관 (r=0.63~0.66)

---

### 2️⃣ 통계 검정 (Statistical Testing)

**하는 일**: 데이터 간의 차이/관계가 우연인지 실제인지 검증

```
┌─────────────────────────────────┐
│        가설 검정 (두 가지)      │
├─────────────────────────────────┤
│ Test 1: t-test                  │
│ Q: 서울과 부산의 평균 매출이    │
│    정말 다를까?                 │
│ A: 통계적으로 차이 없음         │
│    (p-value = 0.467 > 0.05)   │
│                                 │
│ Test 2: 카이제곱 (χ²)          │
│ Q: 지역과 구매 카테고리가       │
│    연관이 있을까?               │
│ A: 유의미한 연관성 있음!        │
│    (p-value = 0.0099 < 0.05)  │
│    → 지역별로 선호 카테고리     │
│       다름!                     │
└─────────────────────────────────┘
```

**의사결정에 활용하는 방법**

```
만약 t-test 결과가 "차이 있음"이면?
→ 지역별 다른 마케팅 전략 필요!

만약 카이제곱 결과가 "연관성 있음"이면?
→ 지역별 맞춤 상품 배치 가능!
```

### 3️⃣ 머신러닝 (ML Pipeline)

**하는 일**: 과거 데이터로 학습하여 미래 판매 예측

```
┌──────────────────────────────────┐
│   Ridge 회귀 예측 모델          │
├──────────────────────────────────┤
│ 입력:                            │
│  • 수량 (quantity)              │
│  • 단가 (unit_price)            │
│  • 고객 나이 (age)              │
│  • 지역 (region)                │
│  • 카테고리 (category)          │
│  • 결제 수단 (payment_method)   │
│          ↓                       │
│  [전처리]                        │
│  • 수치형: 정규화 (StandardScaler)
│  • 범주형: 원-핫 인코딩         │
│          ↓                       │
│  [모델 학습]                     │
│  • Ridge 회귀 (L2 정규화)      │
│  • 80% 학습, 20% 검증          │
│          ↓                       │
│ 출력: R² Score = 0.189          │
│ (모델이 18.9% 설명 가능)        │
└──────────────────────────────────┘
```

**예측 활용**

```python
# 신규 거래 매출 예측
새_거래 = {
    'quantity': 10,
    'unit_price': 50000,
    'customer_age': 35,
    'region': '서울',
    'category': '전자',
    'payment_method': '신용카드'
}
예측_매출 = model.predict([새_거래])  # 대략 500,000원?
```

## ❓ FAQ (자주 묻는 질문)

### Q1. 왜 이 시스템이 필요한가?
**A.** 기존에는:
- 분석가가 수동으로 작업 (수일 소요)
- 일관성 없는 결과
- 비용 높음

이제는:
- 자동 분석 (10초)
- 일관된 고품질 결과
- 확장 가능하고 저렴함

### Q2. 데이터 보안은?
**A.** 
- 로컬 환경에서만 실행 (클라우드 미사용)
- 모든 파일이 로컬 디스크에만 저장
- 외부 전송 없음

### Q3. 정확도는 얼마나 되는가?
**A.** 
- 통계 검정: 95% 신뢰도 (p-value 기반)
- ML 예측: R² = 0.189 (18.9% 설명력)
- 성능 개선 중...

### Q4. 새로운 데이터를 분석하려면?
**A.** 간단합니다:
1. CSV 파일을 `data/raw/` 에 저장
2. 파이썬 스크립트 실행
3. 결과 확인 (10초 소요)

### Q5. 문제가 생기면?
**A.** 
1. `logs/pipeline_execution.log` 확인
2. 모든 에러가 기록됨
3. 기술 담당자에게 로그 전달
